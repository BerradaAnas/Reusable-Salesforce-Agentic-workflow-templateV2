# Skill: Deployment Docs

## Purpose

Generate release readiness documentation with manifest clarity, rollback safety, and approval tracking.

## Procedure

1. Summarize approved change scope.
2. List impacted metadata and manifest components.
3. Document dependencies and deployment order.
4. Create pre-deployment checklist.
5. Create post-deployment validation checklist.
6. Create rollback plan and rollback component list.
7. Record approval gates and approvers.

## Output

- Change summary
- Metadata manifest summary
- Pre-deployment checklist
- Deployment order
- Post-deployment validation
- Rollback plan
- Approval gate record
- Release notes

## Guardrails

- Do not imply deployment is approved unless explicitly stated.
- Do not deploy by default.
