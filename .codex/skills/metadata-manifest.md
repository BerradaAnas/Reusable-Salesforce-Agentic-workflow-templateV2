# Skill: Metadata Manifest

## Goal

Produce a deployable component list or package.xml-style manifest draft from approved changes.

## Rules

- Do not guess names.
- Mark unverified components clearly.
- Separate created, modified, and deleted components.
- Do not include destructive changes unless explicitly approved.

## Output sections

- Objects
- Fields
- Validation Rules
- Apex Classes
- Apex Triggers
- LWCs
- Flows
- Layouts
- Flexipages
- Permission Sets
- Tabs
- Custom Metadata
- Applications
- Dependencies
- Deployment order
- Rollback components
